# MAX Resend From Channel To Chat

Бот для MAX, который автоматизирует комментарии к постам в канале.

## Что делает бот

- Отслеживает новые посты в подключенных каналах.
- Пересылает пост в выбранный чат комментариев.
- Добавляет под постом кнопки:
  - переход в чат комментариев;
  - переход к конкретному сообщению в чате.
- Позволяет админам управлять ботом через `/admin`:
  - подключать/удалять пары `канал -> чат`;
  - настраивать текст кнопок и рекламную ссылку;
  - включать mute по времени;
  - редактировать текст/картинку у сохраненных постов.

## Как работает

- Режим получения событий: **Webhook** (без polling).
- Бот поднимает HTTP endpoint и регистрирует подписку в MAX через `/subscriptions`.
- Конфигурация хранится в `config.json`.

## Быстрый запуск

1. Создайте `.env` на основе `.env.example`.
2. Заполните минимум:
   - `MAX_BOT_TOKEN`
   - `ADMIN_USER_IDS`
   - `WEBHOOK_URL` (публичный `https://.../webhook_8091`)
   - `WEBHOOK_LISTEN=0.0.0.0:8091`
3. Поднимите контейнер:
   ```bash
   docker compose up -d --build
   ```
4. Проверьте health:
   - `GET /health` (через ваш reverse proxy).

## Nginx / Reverse proxy

Пример конфига для проксирования вебхука лежит в `config/`.

Смысл:
- внешний HTTPS endpoint (`443`) -> `127.0.0.1:8091` внутри сервера.

## Переменные окружения

См. `.env.example`.

Основные:
- `MAX_BOT_TOKEN` - токен бота MAX.
- `ADMIN_USER_IDS` - user_id админов через запятую.
- `WEBHOOK_URL` - публичный URL вебхука.
- `WEBHOOK_SECRET` - опциональный секрет webhook.
- `WEBHOOK_LISTEN` - bind адрес и порт приложения.
- `COMMENTS_CHAT_TEXT`, `COMMENTS_MESSAGE_BUTTON_TEXT`, `AD_TEXT`, `AD_URL` - тексты/ссылка кнопок.
